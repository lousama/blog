getOutputStream() has already been called for this response这个问题今天又遇到了。
起因是写一个http服务，在index.jsp里面直接调用了server(request,response)。
具体的原因：jsp编译成servlet之后在函数
_jspService(HttpServletRequest request, HttpServletResponse response)的最后有一段如下代码：

finally {    
      if (_jspxFactory != null)     
          _jspxFactory.releasePageContext(_jspx_page_context);    
}

这里是在释放在jsp中使用的对象，会调用response.getWriter(),因为这个方法是和response.getOutputStream()相冲突的！所以会出现以上这个异常。

要解决只需要在index.jsp调用server(request,response) 加上如下代码即可

out.clear();   
out=pageContext.pushBody();

贴上jsp内容

<%@page import="com.ct.Server"%>  
<%@ page language="java" import="java.util.*" pageEncoding="UTF-8"%>  
<%  
String path = request.getContextPath();  
String basePath = request.getScheme()+"://"+request.getServerName()+":"+request.getServerPort()+path+"/";  
Server.post(request, response);  
out.clear();   
out=pageContext.pushBody();  
%>  

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">  
<html>  
  <head>  
    <base href="<%=basePath%>">  

    <title>index</title>  
    <meta http-equiv="pragma" content="no-cache">  
    <meta http-equiv="cache-control" content="no-cache">  
    <meta http-equiv="expires" content="0">      
    <meta http-equiv="keywords" content="keyword1,keyword2,keyword3">  
    <meta http-equiv="description" content="This is my page">  
    <!-- 
    <link rel="stylesheet" type="text/css" href="styles.css"> 
    -->  


  </head>  

  <body>
