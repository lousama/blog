JVM调优之前戏----JDK命令行工具

  • JAVA
  • JVM
JVM

在JDK的命令行中，一般开发人员最耳熟能详的肯定就是java,javac,javap等常用命令，不过在jdk/bin下还有许多其他的命令行工具，它们被用来监测JVM运行时的状态，下面我们来详细解读一下几个常用的工具以及如何使用。

1.jps

jps是最常用也是最基本的命令，该命令主要用于输出JVM进程状况，例子如下：

jps(JVM Process Status)
有点类似UNIX中的ps命令，可以列出正在运行的虚拟机进程，同时还可以显示出虚拟机运行主类(即main()所在的类)名称以及进程在本机的虚拟机ID–LVMID(Local Virtual Machine Identifier)，功能很简单，但却是使用频率最高的命令行，因为后续的命令都要依据这个命令查出来的LVMID来进行操作。

1.1 jps命令格式 

jps [option]

1.2 option参数及说明

  • -l : 输出主类全名或jar路径
  • -q : 只输出LVMID
  • -m : 输出JVM启动时传递给main()的参数
  • -v : 输出JVM启动时显示指定的JVM参数

其中[option]参数也可以不写。

2.jstat

jstat(JVM statistics Monitoring)是用于监视虚拟机运行时状态信息的命令，它可以显示出虚拟机进程中的类装载、内存、垃圾收集、JIT编译等运行数据。

2.1 格式 

jstat [option] LVMID [interval] [count]

2.2 参数详解

  • [option] : 操作参数
  • LVMID : 本地虚拟机进程ID
  • [interval] : 连续输出的时间间隔
  • [count] : 连续输出的次数

2.3 示例详解

主要针对常用的[option]参数

2.3.1 -class

监视类装载、卸载数量、总空间以及耗费的时间

  • Loaded : 加载class的数量
  • Bytes : class字节大小
  • Unloaded : 未加载class的数量
  • Bytes : 未加载class的字节大小
  • Time : 加载时间

2.3.2 -gc

监视Java堆以及GC的状况

C即Capacity 总容量，U即Used 已使用的容量

  • S0C : survivor0区的总容量
  • S1C : survivor1区的总容量
  • S0U : survivor0区已使用的容量
  • S1C : survivor1区已使用的容量
  • EC : Eden区的总容量
  • EU : Eden区已使用的容量
  • OC : Old区的总容量
  • OU : Old区已使用的容量
  • MC : 方法区即永久代的总容量
  • MU : 方法区已使用的容量
  • CCSC : 压缩类空间总容量
  • CCSU : 压缩类空间已使用的容量
  • YGC : 新生代垃圾回收次数
  • YGCT : 新生代垃圾回收时间
  • FGC : 老年代垃圾回收次数
  • FGCT : 老年代垃圾回收时间
  • GCT : 垃圾回收总消耗时间

2.3.3 -gccapacity

同-gc，不过还会输出Java堆各区域使用到的最大、最小空间

下面只列出与-gc命令有差异的项(后续同理)

  • NGCMN : 新生代占用的最小空间
  • NGCMX : 新生代占用的最大空间
  • OGCMN : 老年代占用的最小空间
  • OGCMX : 老年代占用的最大空间
  • MCMN : 方法区占用的最小空间
  • MCMX : 方法区占用的最大空间
  • CCSMN : 压缩类占用的最小空间
  • CCSMX : 压缩类占用的最大空间

2.3.4 -gcutil

同-gc，不过输出的是已使用空间占总空间的百分比

2.3.5 -gcnew/-gcnewcapcacity

监视新生代GC状况/输出最大最小空间

2.3.6 -gcold/-gcoldcapacity

监视老年代GC状况/输出最大最小空间

2.3.7 -compiler

输出JIT编译过的方法数量耗时等

  • Compiled : 编译数量
  • Failed : 编译失败数量
  • Invalid : 无效数量
  • Time : 编译耗时
  • FailedType : 失败类型
  • FailedMethod : 失败方法的全限定名

2.3.7 -printcompilation

输出已被JIT编译的方法

  • Compiled : 最近被JIT编译的方法数量
  • Size : 最近被JIT编译方法的字节码数量
  • Type : 最近被编译的编译类型
  • Method : 方法的去按限定名

3. jinfo

jinfo(JVM Configuration info)这个命令作用是实时查看和调整虚拟机运行参数。
之前的jps -v口令只能查看到显示指定的参数，如果想要查看未被显示指定的参数的值就要使用jinfo口令
